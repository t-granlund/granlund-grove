### A. Portfolio Chatbot (NEW — fully shipped)

**Architecture:** Client-side RAG. Zero external API calls. Runs entirely in the browser.

**Files:**

```
src/components/site/ChatWidget.tsx          # Floating chat UI
src/lib/chatbot/engine.ts                  # Retrieval + response generation
src/lib/chatbot/knowledge.ts             # Knowledge base aggregator
src/lib/chatbot/knowledge/bio-contact.ts # 15 chunks (bio, availability, contact)
src/lib/chatbot/knowledge/career.ts      # 11 chunks (all 5 employers)
src/lib/chatbot/knowledge/skills.ts      # 28 chunks (deep skill detail)
src/lib/chatbot/knowledge/projects.ts    # 17 chunks (projects + ventures)
```

**How it works:**

1. User types query → tokenized into keywords (stopword-filtered)
2. Keyword matching against 60+ knowledge chunks with topic-synonym expansion
3. Priority-weighted scoring (1-10 scale per chunk)
4. Three-tier response generation:
   - Score > 15: Direct answer
   - Score > 8: Contextual answer with related chunks
   - Score ≤ 8 or no match: Fallback → `hello@tylergranlund.com`

**Key design decisions:**

- **No LLM API** — zero latency, zero cost, works offline
- **Persistent history** via localStorage (survives page refresh)
- **Keyboard accessible** — Escape to close, Tab navigation, focus trapping
- **Click-outside-to-close** + accessible dialog pattern
- **Suggested questions** for idle state (first 3 messages)
- **Source tags** on every assistant response (shows which page the info came from)
- **Typing indicator** (animated dots) for natural feel

**Chatbot UI features:**

- Floating cedar-accent button with unread counter
- Glassmorphism panel (backdrop-blur, card/90 bg)
- Header: "Ask about Tyler" + "Open to opportunities" pulse badge
